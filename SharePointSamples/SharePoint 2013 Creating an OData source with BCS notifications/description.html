This project shows how to use Visual Studio 2012 and SharePoint development tools in Visual Studio 2012 to create an app for SharePoint using Business Connectivity Services (BCS) to expose complex data from an external system.

The main objectives for this sample are:

  • Create a new ASP.NET web application that will be used to host the OData service.
  • Add a new data model that is generated by Visual Studio 2012 to retrieve the database information from the Northwind sample database.
  • Create a data service based on the Northwind data model.
  • Configure security for the service
  • Add service operations to the Northwind service to allow BCS to subscribe and unsubscribe to notifications of changes to the underlying data.

This sample uses the Northwind sample database that is downloadable from http://msdn.microsoft.com/en-us/library/ff851969.aspx. When properly configured, the OData service will then be available for discovery by the Visual Studio 2012 autogeneration tools to create an external content type that can then be used by external lists, Office client applications, and custom client object model calls.

Prerequisites

This sample requires the following:

  • Visual Studio 2012
  • Internet Information Services (IIS)
  • SQL Server or MSDE
  • Northwind sample data

Key components of the sample

The NorthwindOData.zip file includes the following:

  • The Northwind sample database files
  • Visual Studio 2012 project files

Configure the sample

Follow these steps to configure the sample.

  1. Install the Northwind sample data. Follow the installation instructions found at Northwind Database Installation.
  2. Extract the SampleBCSApp.zip file to your hard drive.
  3. Start the simulated OData service. This service is hosted by a local instance of IIS.  It simply attaches to a port in IIS and provides an OData endpoint that you will use in your app.
  4. Open the Visual Studio project.
  5. Build and deploy the sample.
  6. Test the app by creating and modifying records.  

Build the sample

Follow these steps to build the sample.

To build the project

  • Press F5 to build, run and debug the new project.
